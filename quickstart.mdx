This guide walks through building a fully working Trix editor from scratch. By the end you will have an editor that submits its HTML content with a form.

## Minimal working example

<Steps>
  <Step title="Include the CSS and JavaScript">
    Add Trix to your page's `<head>`. The CSS provides default styles for the toolbar and editor; the script registers the `<trix-editor>` and `<trix-toolbar>` custom elements.

    ```html
    <head>
      <link rel="stylesheet" type="text/css" href="https://unpkg.com/trix@2.1.17/dist/trix.css">
      <script type="text/javascript" src="https://unpkg.com/trix@2.1.17/dist/trix.umd.min.js"></script>
    </head>
    ```
  </Step>

  <Step title="Add a hidden input inside your form">
    Trix stores its HTML output in a hidden `<input>` element. Give it an `id` so the editor can reference it, and a `name` so the value is submitted with your form.

    ```html
    <form method="post" action="/posts">
      <input id="post_content" type="hidden" name="content">
    </form>
    ```
  </Step>

  <Step title="Add the trix-editor element">
    Place `<trix-editor>` inside the form and set its `input` attribute to the `id` of the hidden input. Trix will automatically insert a `<trix-toolbar>` immediately before the editor.

    ```html
    <form method="post" action="/posts">
      <input id="post_content" type="hidden" name="content">
      <trix-editor input="post_content"></trix-editor>
    </form>
    ```

    <Tip>
      `<trix-editor>` also accepts `autofocus` and `placeholder` attributes, just like `<textarea>`. Unlike a `<textarea>`, it expands vertically to fit its contents automatically.
    </Tip>
  </Step>

  <Step title="Submit the form">
    Trix keeps the hidden input's `value` in sync with the editor content after every change. When the form is submitted, your server receives the HTML content in the `content` parameter — no extra JavaScript required.

    To read the current value before submission:

    ```javascript
    const input = document.getElementById("post_content")
    console.log(input.value) // HTML string of current editor content
    ```
  </Step>
</Steps>

## Complete minimal HTML

Here is the full example as a single HTML file:

```html
<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" type="text/css" href="https://unpkg.com/trix@2.1.17/dist/trix.css">
    <script type="text/javascript" src="https://unpkg.com/trix@2.1.17/dist/trix.umd.min.js"></script>
  </head>
  <body>
    <form method="post" action="/posts">
      <input id="post_content" type="hidden" name="content">
      <trix-editor input="post_content"></trix-editor>
      <button type="submit">Save</button>
    </form>
  </body>
</html>
```

## Loading existing content

To pre-populate the editor with saved content, set the `value` attribute on the hidden input element. Trix reads the input's initial value when the editor connects to the DOM.

```html
<form method="post" action="/posts/1">
  <input id="post_content" value="<h1>Hello world</h1>" type="hidden" name="content">
  <trix-editor input="post_content"></trix-editor>
</form>
```

<Warning>
  When a `<trix-editor>` connects with both inline HTML content inside its tags and an associated `input` element, Trix always disregards the inline HTML and loads from the input's `value`.
</Warning>

Alternatively, if you are not using an associated `<input>` element, Trix will safely sanitize and load any HTML content placed directly inside the `<trix-editor>` tag:

```html
<trix-editor>Editor content goes here</trix-editor>
```

## Accessing the editor API

Every `<trix-editor>` element exposes an `editor` property once initialized. This is a `Trix.Editor` instance you can use to manipulate the document programmatically.

```javascript
const element = document.querySelector("trix-editor")
const editor = element.editor  // Trix.Editor instance
```

Wait for the `trix-initialize` event before accessing the `editor` property to ensure the element has finished connecting:

```javascript
document.addEventListener("trix-initialize", (event) => {
  const editor = event.target.editor

  // Insert text at the current cursor position
  editor.insertString("Hello, world!")

  // Get the current document as HTML
  const element = event.target
  console.log(element.value) // HTML string
})
```

The `editor` property gives you access to the full programmatic API:

```javascript
// Insert text and HTML
editor.insertString("Hello")
editor.insertHTML("<strong>Bold text</strong>")

// Manage selections
editor.setSelectedRange([0, 5])
editor.getSelectedRange()  // [0, 5]

// Apply formatting to the current selection
editor.activateAttribute("bold")
editor.deactivateAttribute("bold")

// Undo and redo
editor.undo()
editor.redo()

// Inspect the document model
const doc = editor.getDocument()  // Trix.Document instance
doc.toString()  // Plain text string
```

## Custom toolbar placement

By default Trix inserts `<trix-toolbar>` immediately before `<trix-editor>`. To place the toolbar elsewhere, render it yourself and reference its `id` with the `toolbar` attribute:

```html
<main>
  <trix-toolbar id="my_toolbar"></trix-toolbar>
  <div class="other-content"></div>
  <trix-editor toolbar="my_toolbar" input="post_content"></trix-editor>
</main>
```
