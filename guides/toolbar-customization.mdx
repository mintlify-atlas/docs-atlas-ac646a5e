title: "Toolbar Customization"
description: "Override the default toolbar HTML, add custom buttons, respond to built-in and external actions, and wire up keyboard shortcuts."

Trix renders a `<trix-toolbar>` automatically before every editor. The toolbar HTML is generated by `Trix.config.toolbar.getDefaultHTML()`. You can override this function to take full control of what the toolbar contains.

## Overriding the default HTML

Overwrite `Trix.config.toolbar.getDefaultHTML` before any editors initialize. Use the `trix-before-initialize` event to ensure your override runs at the right time:

```javascript
import Trix from "trix"

document.addEventListener("trix-before-initialize", () => {
  Trix.config.toolbar.getDefaultHTML = () => `
    <div class="trix-button-row">
      <span class="trix-button-group" data-trix-button-group="text-tools">
        <button type="button" class="trix-button trix-button--icon trix-button--icon-bold"
          data-trix-attribute="bold" data-trix-key="b" title="Bold" tabindex="-1">Bold</button>
        <button type="button" class="trix-button trix-button--icon trix-button--icon-italic"
          data-trix-attribute="italic" data-trix-key="i" title="Italic" tabindex="-1">Italic</button>
      </span>
    </div>
  `
})
```

<Note>
  `getDefaultHTML` is called once per editor when its toolbar is created. Changes after an editor has initialized will not take effect for that editor.
</Note>

## Button data attributes

Trix reads three data attributes to determine what a toolbar button does.

<CardGroup cols={3}>
  <Card title="data-trix-attribute" icon="tag">
    Toggles a text or block attribute on the current selection. The value must match a key in `Trix.config.textAttributes` or `Trix.config.blockAttributes`.
  </Card>
  <Card title="data-trix-action" icon="bolt">
    Invokes a built-in or custom action by name when the button is clicked.
  </Card>
  <Card title="data-trix-key" icon="keyboard">
    Registers a keyboard shortcut. Trix listens for <kbd>Meta</kbd> (or <kbd>Ctrl</kbd>) plus the specified key. Use `shift+` as a prefix for shift combinations.
  </Card>
</CardGroup>

### Toggle attribute buttons

A button with `data-trix-attribute` toggles formatting on the selected text:

```html
<!-- Inline text attribute (wraps selected text in <strong>) -->
<button type="button" data-trix-attribute="bold" data-trix-key="b">Bold</button>

<!-- Inline text attribute (wraps selected text in <em>) -->
<button type="button" data-trix-attribute="italic" data-trix-key="i">Italic</button>

<!-- Block attribute (wraps the current block in <blockquote>) -->
<button type="button" data-trix-attribute="quote">Quote</button>

<!-- Block attribute (wraps the current block in <h1>) -->
<button type="button" data-trix-attribute="heading1">Heading</button>
```

Trix looks up the attribute name in `Trix.config.textAttributes` first, then `Trix.config.blockAttributes`. The default inline attributes are `bold`, `italic`, `href`, and `strike`. The default block attributes are `heading1`, `quote`, `code`, `bullet`, and `number`.

### Built-in action buttons

A button with `data-trix-action` invokes an internal editor action:

```html
<button type="button" data-trix-action="undo" data-trix-key="z">Undo</button>
<button type="button" data-trix-action="redo" data-trix-key="shift+z">Redo</button>
<button type="button" data-trix-action="link" data-trix-key="k">Link</button>
<button type="button" data-trix-action="attachFiles">Attach Files</button>
<button type="button" data-trix-action="increaseNestingLevel">Indent</button>
<button type="button" data-trix-action="decreaseNestingLevel">Outdent</button>
```

## Custom external actions

Prefix an action name with `x-` to define an external (custom) action. Trix will not handle it internally — instead it fires a `trix-action-invoke` event that you respond to in your own code.

```html
<button id="log-button" type="button" data-trix-action="x-log">Log</button>
```

```javascript
document.addEventListener("trix-action-invoke", function(event) {
  const { target, invokingElement, actionName } = event

  if (actionName === "x-log") {
    console.log(
      `Custom ${actionName} invoked from ${invokingElement.id} button on ${target.id} trix-editor`
    )
  }
})
```

The `trix-action-invoke` event exposes three properties:

| Property | Description |
|---|---|
| `target` | The `<trix-editor>` element |
| `invokingElement` | The `<button>` that was clicked |
| `actionName` | The value of `data-trix-action` (e.g. `"x-log"`) |

<Note>
  `trix-action-invoke` only fires for custom `x-` actions, not for built-in actions like `undo` or `redo`.
</Note>

## Button groups

Group related buttons inside a `<span>` with the `trix-button-group` class and a `data-trix-button-group` attribute. Trix uses the `trix-button-group-spacer` element to push the history group to the far right:

```html
<div class="trix-button-row">
  <span class="trix-button-group" data-trix-button-group="text-tools">
    <!-- text buttons -->
  </span>
  <span class="trix-button-group" data-trix-button-group="block-tools">
    <!-- block buttons -->
  </span>
  <span class="trix-button-group-spacer"></span>
  <span class="trix-button-group" data-trix-button-group="history-tools">
    <!-- undo / redo buttons -->
  </span>
</div>
```
